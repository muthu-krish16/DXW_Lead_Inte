# DXW Lead Intelligence

AI-powered B2B lead research tool for DataXWorks. Upload a prospect list, select a company, and get a 15-section intelligence report — business operations, tech stack, data architecture, hiring signals, pain chains, funding, scores, and a DXW opportunity map — generated by Gemini from live web scraping and search.

---

## What It Does

For each company in your Excel file, the tool:

1. Scrapes the company's website and auto-crawls up to 15 internal pages
2. Runs 14 DuckDuckGo search queries (news, tech stack, funding, Glassdoor, Crunchbase, etc.)
3. Detects the live tech stack from HTML signatures
4. Sends all gathered content to Gemini in 3 structured API calls
5. Returns a merged 15-section JSON report displayed in Streamlit
6. Saves every report to a local SQLite database

---

## Project Structure

```
├── app.py           # Streamlit UI — upload, select, display
├── analyzer.py      # Scraping, search, tech detection, Gemini calls
├── scoring.py       # Weighted final score (0–100%) from section 14
├── database.py      # SQLite schema, save/query helpers
├── prompt.txt       # DXW profile + rules + JSON output schema (required)
└── lead_intelligence.db  # Auto-created on first run
```

## Setup

**Python 3.10+ required.**

```bash
pip install streamlit pandas google-generativeai requests \
            beautifulsoup4 duckduckgo-search openpyxl
```

`prompt.txt` must exist in the project root. It defines the DXW service profile, analysis rules, and the expected JSON output structure that Gemini follows.

---

## Running

```bash
streamlit run app.py
```

Then open `http://localhost:8501` in your browser.

## Input Excel Format

The Excel file must have at least these columns (header names are case-sensitive after stripping whitespace):

| Column | Description |
|---|---|
| `company_name` | Used to group rows per company |
| `Domain` | Primary website URL |
| `LinkedIn_Company` | LinkedIn company page (optional) |
| `Blog` | Blog URL (optional) |
| `News` | News/press page (optional) |
| `First Name` / `Last Name` | Contact names |
| `Title` | Contact job title |
| `Email ID` | Contact email |
| `industry` | Industry label |
| `Employee Size` | Headcount band |
| `Revenue` | Revenue band |
| `City` / `State` / `Country` | Location fields |

Multiple rows with the same `company_name` are treated as contacts for the same company. The first row supplies all company-level fields.

## Usage

1. Paste your **Gemini API key** in the key field at the top
2. Upload your `.xlsx` file
3. Select a company from the dropdown
4. Click **🚀 Analyze This Company**
5. Wait for the 5-step pipeline to complete (typically 60–120 seconds)
6. Scroll through the 15-section report

The **🗑️ Clear Memory** button in the analysis view clears Streamlit session state while preserving your API key.

## Scoring

Final score (0–100%) is a weighted average of the 8 dimension scores from section 14:

| Dimension | Weight |
|---|---|
| DXW Fitment | 20% |
| Operational Complexity | 15% |
| Data Architecture Risk | 15% |
| Governance Gap | 15% |
| Tech Modernization Need | 10% |
| AI Maturity | 10% |
| Timing / Urgency | 10% |
| Account Potential | 5% |

Tier is read from section 15 summary (`Tier 1 / 2 / 3`). If missing, it falls back to score thresholds: ≥75 → Tier 1, ≥50 → Tier 2, else Tier 3.

## Database

Reports are auto-saved to `lead_intelligence.db` (SQLite) after every successful analysis. The schema has two tables:

- `reports` — one row per analysis with all scores, summary fields, and the full report JSON
- `contacts` — one row per contact, linked to the report by `report_id`

The `database.py` module also exposes `get_all_reports()` and `get_report_by_id()` for building a history view or export feature.

> **Note:** `create_tables()` drops and recreates both tables on every app start to avoid schema drift. Do not store data you need to keep across code changes without backing up the `.db` file first.

## Gemini API Notes

- Model used: `gemini-2.0-flash`
- 3 API calls per analysis (sections 1–4, 5–8, 9–15)
- Each call has a 120-second timeout and up to 3 retry attempts with exponential backoff
- Prompts are capped at 150,000 characters to avoid latency-induced 504s
- Get a free API key at [aistudio.google.com](https://aistudio.google.com)

## Known Limitations

- Some sites block scraping (bot protection, Cloudflare). The pipeline handles failures gracefully and continues with whatever content was collected.
- LinkedIn URLs are excluded from crawling (robots.txt / login wall).
- DuckDuckGo rate-limits aggressive searches; failed queries are skipped silently.
- The database is local only — no multi-user support.
